# 订单数据验证需求清单

**目的**: 将E2E报告中依赖D3 Meltwater单源的推断性结论，通过订单数据进行交叉验证和降级升级。

**生成日期**: 2026-05-20
**关联报告**: VOC喂养电器E2E洞察报告 V5

---

## 一、所需数据维度与口径

### A. 用户画像验证

| # | 需求数据 | 口径 | 验证什么结论 | 优先级 |
|---|---------|------|------------|--------|
| A1 | 购买用户人口统计（年龄/性别/地区）| 按品类（消毒器/暖奶器/吸奶器）拆分 | 颜值生活型是否真的是暖奶器最大层（4,901条, D3单源） | P0 |
| A2 | 用户首次购买品类分布 | 独立首购用户数据 | 消毒器首购占比17.08%（D4已有但需持续验证） | P0 |
| A3 | 用户交叉购买路径 | 购买序列数据（先买A后买B）| 消毒器→吸奶器反向连带15.76%，正向连带4.93% | P0 |
| A4 | CRM标签/用户分层 | 与D3的8层画像标签对应关系 | 安全焦虑驱动型是消毒器核心层的准确性 | P1 |

### B. 场景验证

| # | 需求数据 | 口径 | 验证什么结论 | 优先级 |
|---|---------|------|------------|--------|
| B1 | 订单时间分布（按小时）| 按品类拆分 | 夜间喂养是暖奶器第一场景（2,679条, D3单源）| P1 |
| B2 | 退货时间分布 | 按品类+小时拆分 | "凌晨低脑力失败场景"是否导致退货 | P1 |
| B3 | 月度销售数据（完整年度）| 2025-2026全量 | 3-4月旺季假设（目前仅3个月D3数据） | P0 |
| B4 | 季节性指数 | 按品类+区域 | 验证"共同旺季"结论，区分品类间差异 | P1 |

### C. 满意度/质量验证

| # | 需求数据 | 口径 | 验证什么结论 | 优先级 |
|---|---------|------|------------|--------|
| C1 | 退货原因分类 | 按品类+原因标签 | 温控失准是否暖奶器退货第一原因（D6: 50,117条） | P0 |
| C2 | 暖奶器退货率趋势 | 按月+按SKU | 验证L1=12.59的严重性和趋势 | P0 |

| C4 | NPS/CSAT调查数据 | 标准NPS问卷（非Proxy NPS） | 替代当前的Proxy NPS口径 | P1 |

### D. 渠道/平台验证

| # | 需求数据 | 口径 | 验证什么结论 | 优先级 |

| D1 | 各渠道流量来源 | GA/UTM + 订单归因 | 消毒器是否需要独立获客（非依赖吸奶器导流） | P1 |
| D2 | 社媒投放ROAS（分品类+平台）| Meta/Google/TikTok | 暖奶器内容孤岛假设 | P1 |

## 二、数据获取优先级矩阵

```
                      验证价值
                    高      中      低
                ┌───────┬───────┬───────┐
获取  容易      │ A2,A3  │ C3    │ D1    │
难度  中等      │ A1,B3  │ B1,B2 │ D2,D3 │
      困难      │ C1,C2  │ A4,C4 │ B4    │
                └───────┴───────┴───────┘
```

## 三、P0 优先获取项（立即执行）

| # | 数据项 | 获取方式 | 预期产出 |
|---|-------|---------|---------|
| 1 | 用户交叉购买路径 | 订单数据库SQL查询 | 验证反向连带15.76%，输出消毒器获客独立性评估 |
| 2 | 暖奶器退货原因+率 | 售后工单系统导出 | 验证温控失准是否P0退货原因 |
| 3 | 消毒器退货原因 | 售后工单系统导出 | 验证烘干不彻底是否P0退货原因 |
| 4 | 完整月度销售数据 | 经营数据库导出 | 验证3-4月旺季假设，输出完整季节性模型 |
| 5 | 购买用户人口统计 | CRM系统导出 | 验证8层画像标签准确性 |

## 四、验证成功后的动作

- **A1验证通过** → 用户层数据从"推断"升级为"确认"，去掉D3单源标注
- **B3验证通过** → 季节性结论升级，加入年度趋势图
- **C1-C3验证通过** → P0优先级从"三源一致"升级为"四源一致"（+订单数据）
- **验证失败** → 标注为"D3数据与订单数据矛盾，需进一步调查"，暂停基于该结论的策略建议
